# Документация по оплате

## 1. Текущая модель оплаты

Сейчас Сервис использует разовую оплату доступа на 30 дней через YooKassa.

Текущая логика:

1. Пользователь доходит до платного ограничения.
2. Бот создает платеж в YooKassa.
3. YooKassa возвращает `confirmation_url`.
4. Бот показывает пользователю кнопку оплаты.
5. После успешной оплаты YooKassa отправляет webhook.
6. Сервис проверяет платеж через YooKassa API.
7. Если платеж успешный, пользователю выдается или продлевается подписка на 30 дней.

## 2. Тарифы

### Базовый тариф

- Название: доступ к наставнику на 30 дней.
- Стоимость: **299 ₽**.
- Срок доступа: **30 дней**.
- Внутренний код тарифа: `monthly_299`.

### Первый месяц для нового пользователя

- Стоимость первого месяца: **149 ₽**.
- Срок доступа: **30 дней**.
- Применяется только к первому успешному платежу пользователя.
- Новым пользователем считается пользователь, у которого еще не было успешной оплаты.
- После первого оплаченного периода дальнейший доступ стоит **299 ₽ за 30 дней**.

Рекомендуемый внутренний код тарифа для первого месяца: `monthly_first_149`.

## 3. Что считается успешной оплатой

Успешной оплатой считается платеж, который:

- найден в таблице `payments`;
- подтвержден YooKassa через webhook;
- дополнительно проверен через YooKassa API;
- имеет событие `payment.succeeded`;
- имеет `paid=true`;
- переведен в статус `succeeded` в базе Сервиса.

## 4. Как определять право на скидку

Пользователь имеет право на цену первого месяца, если в базе нет ни одного платежа этого пользователя со статусом `succeeded`.

Пример правила:

```text
если successful_payments_count == 0:
    цена = 149 ₽
    tariff_code = monthly_first_149
иначе:
    цена = 299 ₽
    tariff_code = monthly_299
```

Важно: pending/canceled платежи не должны сжигать скидку. Скидка считается использованной только после успешного платежа.

## 5. Paywall-текст

Рекомендуемый текст для первого платежа:

```text
Чтобы продолжить путь с наставником, нужен доступ на месяц.

Для первого месяца я сделал мягкий вход: 149 ₽ вместо 299 ₽.
Так ты спокойно попробуешь формат и поймешь, подходит ли тебе такой темп.

После первого месяца доступ стоит 299 ₽/мес.
```

Рекомендуемый текст для обычного платежа:

```text
Чтобы я вел тебя дальше, нужен доступ на месяц.

Стоимость доступа — 299 ₽ за 30 дней.
После оплаты продолжим с того места, где ты остановился.
```

## 7. Автопродление

Автопродление пока не включено.

Текущая схема создает обычный redirect-платеж и не сохраняет способ оплаты пользователя. Поэтому регулярные списания нельзя включить одной маленькой правкой.

Для автопродления понадобятся отдельные изменения:

- проверить, разрешены ли автоплатежи в магазине YooKassa;
- явно показать пользователю условия регулярных списаний;
- получить согласие пользователя на сохранение способа оплаты;
- создать первый платеж с параметрами сохранения payment method;
- сохранить `payment_method_id` после успешной оплаты;
- добавить поля в базу для автопродления;
- добавить планировщик регулярных списаний;
- обработать неуспешные списания;
- добавить команду или кнопку отмены автопродления;
- обновить пользовательское соглашение и paywall.

## 8. Что не храним

Сервис не должен хранить:

- полный номер банковской карты;
- CVV/CVC;
- срок действия карты;
- платежные данные, которые должен хранить только платежный провайдер.

Сервис может хранить:

- ID платежа YooKassa;
- статус платежа;
- сумму;
- валюту;
- ссылку на оплату;
- дату создания платежа;
- дату успешной оплаты;
- ID сохраненного способа оплаты, если автопродление будет внедрено позднее и пользователь даст согласие.

## 9. Проверки после изменений оплаты

После любой правки оплаты нужно выполнить:

```bash
cd /opt/project && python -m compileall app.py bot services db
```

```bash
cd /opt/project && systemctl restart mentorbot.service
```

```bash
cd /opt/project && systemctl status mentorbot.service --no-pager
```

```bash
cd /opt/project && curl -sS http://127.0.0.1:8000/health
```

При необходимости:

```bash
cd /opt/project && journalctl -u mentorbot.service -n 80 --no-pager
```

```bash
cd /opt/project && tail -n 120 logs/error.log
```
