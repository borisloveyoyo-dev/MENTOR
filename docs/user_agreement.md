# Пользовательское соглашение

> Черновик документа для Telegram-бота-наставника. Перед публикацией нужно заполнить реквизиты владельца сервиса и при необходимости показать юристу.

## 1. Общие положения

1.1. Настоящее Пользовательское соглашение регулирует порядок использования Telegram-бота-наставника, далее — «Сервис».

1.2. Сервис помогает пользователю выбрать направление, получить небольшие практические шаги, возвращаться к действию после пауз и отслеживать прогресс.

1.3. Используя Сервис, пользователь подтверждает, что ознакомился с настоящим Соглашением и принимает его условия.

1.4. Владелец Сервиса: **[указать ИП/самозанятого/ООО/ФИО, ИНН, контактный email]**.

1.5. Сервис работает внутри Telegram. Использование Telegram регулируется отдельными правилами Telegram.

## 2. Что предоставляет Сервис

2.1. Сервис предоставляет пользователю доступ к функциям наставника:

- прохождение анкеты;
- подбор возможного направления;
- генерация практических шагов;
- проверка выполненных заданий;
- напоминания и follow-up сообщения;
- отображение прогресса.

2.2. Сервис не является образовательной организацией, медицинской, психологической, юридической, финансовой или карьерной консультацией в профессиональном смысле.

2.3. Рекомендации Сервиса носят информационный и поддерживающий характер. Пользователь самостоятельно принимает решения и отвечает за свои действия.

## 3. Регистрация и начало использования

3.1. Для использования Сервиса пользователь запускает Telegram-бота и проходит вводный сценарий.

3.2. Сервис может сохранять Telegram ID пользователя, username, имя, ответы анкеты, выбранное направление, историю заданий, результаты проверок и данные о платежах.

3.3. Пользователь обязуется не отправлять в Сервис незаконные материалы, персональные данные третьих лиц без разрешения, спам, вредоносные ссылки и контент, нарушающий права других лиц.

## 4. Платный доступ

4.1. Часть функций Сервиса может быть доступна только после оплаты.

4.2. Базовая стоимость доступа: **299 ₽ за 30 дней**.

4.3. Для нового пользователя может применяться специальная цена первого месяца: **149 ₽ за первые 30 дней**.

4.4. Новым пользователем считается пользователь, у которого ранее не было успешной оплаты доступа к Сервису.

4.5. Специальная цена применяется только к первому успешному платежу пользователя. После окончания первого оплаченного периода дальнейший доступ предоставляется по базовой стоимости, если иное прямо не указано в интерфейсе Сервиса.

4.6. Оплата проводится через платежного провайдера YooKassa. Сервис не хранит полные данные банковских карт пользователя.

4.7. Доступ активируется после получения Сервисом подтверждения успешной оплаты от платежного провайдера.

## 5. Автопродление

5.1. На момент публикации этого черновика доступ оплачивается разовыми платежами на 30 дней.

5.2. Автоматическое продление подписки может быть добавлено позднее только при явном согласии пользователя.

5.3. Если автопродление будет включено, Сервис обязан заранее показать пользователю условия регулярных списаний: сумму, периодичность, способ отмены и момент следующего списания.

5.4. Пользователь должен иметь возможность отказаться от автопродления.

## 6. Возвраты

6.1. Пользователь может обратиться по вопросу возврата по адресу: **[указать email поддержки]**.

6.2. Запрос рассматривается индивидуально с учетом факта оплаты, срока использования доступа и технических обстоятельств.

6.3. Если доступ не был активирован из-за технической ошибки на стороне Сервиса, владелец Сервиса либо активирует доступ, либо возвращает оплату.

6.4. Если пользователь начал пользоваться оплаченным доступом, возврат может быть ограничен с учетом фактически оказанной услуги и применимого законодательства.

## 7. Ограничения ответственности

7.1. Сервис помогает пользователю двигаться через небольшие практические шаги, но не гарантирует конкретный доход, трудоустройство, творческий успех или иной внешний результат.

7.2. Пользователь понимает, что результат зависит от его действий, регулярности, исходных условий и внешних обстоятельств.

7.3. Сервис может временно быть недоступен из-за технических работ, сбоев Telegram, платежного провайдера, хостинга или иных внешних сервисов.

## 8. Персональные данные

8.1. Сервис обрабатывает данные пользователя для работы бота, персонализации заданий, учета прогресса, обработки платежей и технической поддержки.

8.2. Пользователь может запросить удаление своих данных через команду `/delete_me`, если такая команда доступна в боте, либо через контакт поддержки: **[указать email поддержки]**.

8.3. Подробный порядок обработки данных описан в Политике конфиденциальности: `docs/privacy_policy.md`.

## 9. Изменение условий

9.1. Владелец Сервиса может обновлять настоящее Соглашение.

9.2. Новая редакция применяется с момента публикации, если в ней не указан иной срок.

9.3. Если изменения существенно затрагивают платный доступ, пользователь должен быть уведомлен разумным способом через интерфейс Сервиса или сообщение в Telegram.
