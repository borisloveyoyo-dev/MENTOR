# Политика конфиденциальности

> Черновик документа для Telegram-бота-наставника. Перед публикацией нужно заполнить реквизиты владельца сервиса, контактный email и при необходимости показать документ юристу.

## 1. Общие положения

1.1. Настоящая Политика конфиденциальности описывает, какие данные обрабатывает Telegram-бот-наставник, далее — «Сервис», зачем они нужны и как пользователь может запросить их удаление.

1.2. Владелец Сервиса: **[указать ИП/самозанятого/ООО/ФИО, ИНН, контактный email]**.

1.3. Используя Сервис, пользователь соглашается с обработкой данных в объеме, необходимом для работы бота, персонализации заданий, учета прогресса, обработки платежей и технической поддержки.

1.4. Сервис работает внутри Telegram. Telegram может самостоятельно обрабатывать данные пользователя по своим правилам и политике конфиденциальности.

## 2. Какие данные может обрабатывать Сервис

Сервис может обрабатывать следующие данные:

- Telegram ID пользователя;
- username в Telegram;
- имя и фамилию из Telegram-профиля, если они доступны;
- имя, которое пользователь сам указал в боте;
- ответы на вопросы анкеты;
- выбранное направление;
- данные о текущих и выполненных заданиях;
- текстовые сообщения пользователя;
- голосовые сообщения, фото или другие материалы, если пользователь отправляет их для проверки задания;
- результаты AI-проверки и краткие выводы по прогрессу;
- технические логи работы бота;
- данные о платежах: ID платежа YooKassa, сумма, валюта, статус, дата создания и дата успешной оплаты.

## 3. Какие платежные данные не хранит Сервис

Сервис не хранит:

- полный номер банковской карты;
- CVV/CVC;
- срок действия карты;
- данные, необходимые для самостоятельного списания денег с карты.

Оплата проходит через платежного провайдера YooKassa. Платежные данные обрабатываются на стороне платежного провайдера.

## 4. Зачем Сервис обрабатывает данные

Данные используются для следующих целей:

- запуск и работа Telegram-бота;
- прохождение onboarding-анкеты;
- подбор направления;
- генерация практических шагов;
- проверка выполненных заданий;
- отображение прогресса;
- отправка follow-up сообщений и напоминаний;
- обработка оплаты и выдача доступа;
- защита от ошибок, дублей платежей и технических сбоев;
- поддержка пользователя при обращениях.

## 5. AI-обработка данных

5.1. Для генерации ответов, заданий, проверок и кратких выводов Сервис может передавать часть пользовательских сообщений и контекста во внешние AI-сервисы.

5.2. В AI-запросы могут попадать ответы анкеты, выбранное направление, описание задания, присланный пользователем результат и история взаимодействия, необходимая для осмысленного ответа.

5.3. Пользователь не должен отправлять в Сервис чувствительные данные, которые не нужны для работы наставника: паспортные данные, банковские реквизиты, медицинские документы, пароли, коды доступа, данные третьих лиц без их согласия.

## 6. Хранение данных

6.1. Данные хранятся столько, сколько необходимо для работы Сервиса, учета доступа, поддержки пользователя и соблюдения законных обязанностей владельца Сервиса.

6.2. Технические логи могут храниться отдельно от основной базы и использоваться для диагностики ошибок, безопасности и стабильности работы.

6.3. Данные о платежах могут храниться дольше пользовательских сообщений, если это необходимо для учета оплат, возвратов, разрешения спорных ситуаций и выполнения требований законодательства.

## 7. Передача данных третьим лицам

Сервис может передавать данные следующим категориям получателей:

- Telegram — для доставки сообщений и работы бота;
- YooKassa — для обработки платежей;
- AI-провайдеры — для генерации ответов, заданий и проверки результатов;
- хостинг-провайдеры и технические сервисы — для размещения и стабильной работы Сервиса;
- государственные органы — если такая передача требуется по закону.

Сервис не продает пользовательские данные рекламодателям.

## 8. Удаление данных

8.1. Пользователь может запросить удаление своих данных через команду `/delete_me`, если она доступна в боте.

8.2. Также пользователь может направить запрос на удаление данных по адресу: **[указать email поддержки]**.

8.3. После удаления пользовательских данных Сервис может сохранить минимальную техническую и платежную информацию, если это необходимо для учета оплат, возвратов, защиты от злоупотреблений или выполнения требований законодательства.

## 9. Безопасность

9.1. Владелец Сервиса принимает разумные технические меры для защиты данных от несанкционированного доступа, потери и изменения.

9.2. Доступ к серверу, базе данных, webhook-адресам и платежным настройкам должен быть ограничен.

9.3. Пользователь понимает, что ни один способ передачи и хранения данных в интернете не может быть абсолютно безопасным.

## 10. Права пользователя

Пользователь может:

- запросить информацию о своих данных;
- запросить удаление данных;
- прекратить использование Сервиса;
- обратиться по вопросам оплаты, доступа и обработки данных.

Контакт для обращений: **[указать email поддержки]**.

## 11. Изменение Политики

11.1. Владелец Сервиса может обновлять настоящую Политику конфиденциальности.

11.2. Новая редакция применяется с момента публикации, если в ней не указан иной срок.

11.3. Если изменения существенно затрагивают обработку данных пользователя, владелец Сервиса должен уведомить пользователя разумным способом через интерфейс Сервиса или сообщение в Telegram.

## 12. Контакты
